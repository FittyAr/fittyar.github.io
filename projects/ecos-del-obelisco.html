Ecos del Obelisco

Un videojuego de misterio y exploración atmosférica en desarrollo activo, diseñado para aplicar patrones avanzados de arquitectura y desacoplamiento de sistemas en Godot y C#.

El Concepto y la Ingeniería

Ecos del Obelisco es un videojuego que explora mecánicas de misterio y exploración interactiva. Aunque es un proyecto de hobby y desarrollo paralelo, está estructurado con el mismo rigor y disciplina de ingeniería de software que un producto corporativo, evitando el acoplamiento rígido de scripts para garantizar que el proyecto permanezca modular y mantenible a medida que crece.

Arquitectura del Gameplay

Para resolver la lógica del juego, se han implementado patrones de diseño robustos:

  • Arquitectura Basada en Componentes: La lógica del jugador y los sistemas interactivos del mapa se implementan como nodos independientes que se comunican exclusivamente por señales y eventos, eliminando referencias cruzadas directas.
  • Desarrollo en C# (.NET 8/9): Toda la lógica central y las mecánicas avanzadas se desarrollan en C# para aprovechar la tipificación estática, genéricos y rapidez de procesamiento del CLR de .NET.
  • Máquinas de Estado Jerárquicas (HFSM): El control del jugador y el comportamiento de la inteligencia artificial de los objetos se estructuran mediante un patrón Estado orientado a nodos en Godot.
C# Code — Godot 4 State Pattern Example 
using Godot;

public partial class PlayerStateMachine : Node
{
    [Export] public CharacterBody2D Actor { get; set; }
    private PlayerState _currentState;

    public override void _Ready()
    {
        // Inicializar con el estado Idle hijo del nodo
        TransitionTo(GetNode<PlayerState>("Idle"));
    }

    public override void _PhysicsProcess(double delta)
    {
        _currentState?.Update(delta);
    }

    public void TransitionTo(PlayerState newState)
    {
        if (newState == null) return;
        _currentState?.Exit();
        _currentState = newState;
        _currentState.Enter();
    }
}

Gestión Empresarial

Un caso de estudio sobre el diseño e implementación de un sistema moderno de administración corporativa desarrollado en .NET 9 usando Blazor WebAssembly y .NET MAUI Blazor Hybrid con base de datos PostgreSQL.

Caso de Estudio & Contexto Técnico

Este desarrollo es una solución integral y modular diseñada para digitalizar la administración de una organización. El proyecto está estructurado bajo una arquitectura limpia (Clean Architecture), dividiendo estrictamente las capas de dominio, aplicación e infraestructura, lo que permite compartir la base del código y los componentes de interfaz de usuario de Blazor en múltiples plataformas (Web y Windows).

Arquitectura de Software

La aplicación promueve el desacoplamiento de capas mediante CQRS, inyección de dependencias y mapeos de datos limpios:

  • Frontend WebAssembly: Interfaz SPA fluida y reactiva montada en el navegador con Blazor WASM y MudBlazor, optimizando los recursos del servidor.
  • Aplicaciones Nativas (Hybrid): Distribución de escritorio mediante .NET MAUI Blazor Hybrid, compartiendo el 100% de la lógica de vistas y servicios.
  • Seguridad de Acceso Dinámica: Motor de autorización basado en ámbitos de permisos jerárquicos (Global, Unidad, Compañía) evaluados en tiempo de ejecución.
  • Base de Datos y Persistencia: Repositorio persistido en PostgreSQL 18 empleando Entity Framework Core, con soporte transaccional y seeds de datos estructurados.

Módulos Funcionales Implementados

El sistema se compone de varios módulos de negocio interconectados:

  • Módulo de Personal: Registro de empleados, vinculación con cuentas de identidad de ASP.NET Core Identity y subida segura de imágenes en Base64.
  • Parque Automotor & Taller: Administración detallada de la flota vehicular, control de pólizas de seguro, historial de kilometraje/horómetros e integración con órdenes de trabajo de mantenimiento.
  • Sanidad: Agendas médicas con control de privacidad, asignación de turnos, y un asistente de cálculo de riesgo cardiovascular basado en el modelo HEARTS de la OMS.
  • Informatica: Planificación y seguimiento de tareas organizadas a través de calendarios interactivos con soporte drag-and-drop.
  • Mantenimiento y Tickets: Flujo unificado de solicitudes de soporte de infraestructura y triage pre-hospitalario (TPH), con control de costos de materiales y SLAs automáticos.
  • Reportes y Exportación: Motor dinámico de reportes que interpreta filtros en formato JSON, construye consultas con Dynamic LINQ en base de datos y genera archivos Excel (ClosedXML) y PDF (QuestPDF) listos para descargar.

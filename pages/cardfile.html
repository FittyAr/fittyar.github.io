Utilidad Retro

Cardfile

Gestor de tarjetas y archivos multiplataforma inspirado en la clásica utilidad de Windows, originalmente desarrollado en VB.NET y hoy reescrito desde cero con Python 3.10+ y Flet (Flutter para Python).

Stack del Proyecto

Tecnologías Usadas

Plataforma Python 3.10+ & Docker
Framework UI Flet 1.0 Beta (Flutter for Python)
Localización Soporte nativo para 7 idiomas

Características Principales

Combina la rapidez del CRUD clásico con la potencia de Markdown y la portabilidad moderna.

Inspiración Clásica

Inspirado en el objetivo y funcionamiento de la clásica aplicación Cardfile de Windows, pero adaptado a una experiencia de usuario moderna, limpia y fluida.

Editor Markdown

Carga, edita y gestiona el contenido de tus tarjetas utilizando formato Markdown, con un visualizador interactivo y previsualización en vivo.

Soporte Multilingüe

Traducido y localizado nativamente para 7 idiomas: Español, Inglés, Portugués (BR), Francés, Alemán, Ruso y Chino.

Funcionalidades Avanzadas

Detalles técnicos e interactivos pensados para la usabilidad y la seguridad de los datos.

Seguridad y Privacidad

Acceso protegido mediante autenticación con cifrado y hashing de contraseñas, garantizando que el contenido de las tarjetas esté a salvo.

Papelera de Reciclaje

Ciclo de vida completo del contenido con estados de eliminación lógica y un contenedor de recuperación para restaurar tarjetas borradas.

Multiplataforma

Ejecución nativa en escritorio para Windows, macOS y Linux, así como despliegue web interactivo directamente en el navegador.

Instalación y Despliegue

Pasos sencillos para correr Cardfile de forma local o a través de contenedores.

1. Ejecución Local (Python)

Crea un entorno virtual, instala las dependencias y lanza el script principal:

# Crear y activar entorno virtual
python -m venv venv
source venv/bin/activate  # En Windows: .\venv\Scripts\activate

# Instalar dependencias
pip install --upgrade pip
pip install -r requirements.txt

# Ejecutar la aplicación
python main.py

2. Ejecución con Docker

Si prefieres correr la aplicación de forma aislada sin configurar Python:

cd docker
docker-compose up --build

Una vez iniciado el contenedor, abre tu navegador e ingresa a http://localhost:8550.
